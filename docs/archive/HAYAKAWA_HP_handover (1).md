## 2. HPの位置づけと目的

### 2.1 HPの役割

このHPは**売り込みのLPではない**。以下の役割を持つ中継地点として設計する。

- Instagram、GBP（Googleビジネスプロフィール）と連携する情報の母体
- 各種SNS・外部サービスへのハブ（中継地点）
- お店に興味を持った人が雰囲気を感じる場所

### 2.2 流入経路の想定

- **想定する流入**: Instagram、GBP、口コミなど（すでに興味を持った人）
- **想定しない流入**: オーガニック検索（SEO目的ではない）

### 2.3 この位置づけから導かれる設計方針

- 情報を詰め込まない
- ビジュアル・雰囲気を前面に
- 詳細情報（営業時間詳細、地図など）はGBPに任せてリンクで誘導
- 売り込み感・煽り感を出さない

## 3. お店のコンセプトとトーン

### 3.1 お店の特徴

**ハード面**
- 外装: すっきりしたモダンなビジュアル
- 壁: モルタル
- カウンター: 天然木
- 照明: ランプシェード（天井吊り下げ）
- 全体印象: カフェのような落ち着いた空間、かなりおしゃれ

**ソフト面**
- 女性マスターが一人で切り盛り
- カウンターでマスターとの会話を楽しめる
- 女性一人でも入りやすい
- お子様連れ歓迎（ベビーカーOK、子供用椅子あり）
- 地元食材を使用、地域に根ざした餃子づくり
- 週4日営業のぶん丁寧に用意

**餃子の特徴**
- 一口サイズ
- 薄皮でパリッと焼き上げ
- 定番3種: ニラにんにく、ショウガ、シソ
- 月替わりの限定餃子あり

### 3.2 「こうではない」の定義（重要なニュアンス）

以下のイメージは**避ける**：

- ガツガツ系餃子屋
- サッと食べてパッと出る店
- 一般的な餃子屋のイメージ（赤提灯、ビール＆餃子、大衆感）
- 激ウマ！ボリューム満点！的な煽り表現
- 赤・黄色などの派手な配色

### 3.3 目指すイメージ

- 落ち着いている
- すっきり、ミニマル
- コミュニケーション重視（会話を楽しむ場）
- 女性的な柔らかさ
- カフェのような空気感
- 「餃子屋っぽくない餃子屋」

### 3.4 来てほしい人

- コミュニケーションしたい人
- ゆったり過ごしたい人
- 女性一人でも安心して入りたい人
- お子様連れのファミリー

## 5. 確定したページ構成

### 5.1 全体構造

```

├─────────────────────────────────┤

└─────────────────────────────────┘

└─────────────────────────────────┘
```

### 5.2 各セクション詳細

#### セクション1: ファーストビュー

| 項目 | 内容 |
|------|------|
| ビジュアル | 外装（お店のフロント） |
| 要素 | ロゴ（HAYAKAWA）+ 「一口餃子専門店」 |
| 任意要素 | 短いタグライン（検討中） |
| トーン | すっきり、ミニマル、期待感 |

**設計意図**:
- 外装をFVに使う理由: すっきりした見た目で世界観を確立
- 「餃子屋であることが一瞬で伝わりにくい」は問題なし（オーガニック検索想定していない）
- 「食欲を刺激するビジュアルではない」も問題なし（ガツガツ系ではないため）
- ただしテキストで「一口餃子専門店」は入れる（餃子屋であることは伝えたい）

#### セクション2: 店内の空気感

| 項目 | 内容 |
|------|------|
| ビジュアル | 店内写真 |
| 見せたいもの | モルタル壁、天然木カウンター、ランプシェード |
| コピー例 | 「餃子屋っぽくない、餃子屋。」 |
| 枚数 | 1〜2枚程度 |

**設計意図**:
- おしゃれな空間であることを視覚的に伝える
- カウンター席の雰囲気が伝わると良い
- 説明しすぎず、写真で語る

#### セクション3: 今月の餃子

| 項目 | 内容 |
|------|------|
| 内容 | 月替わり限定メニュー |
| 要素 | 写真 + 商品名 + 説明 + 「◯月限定」ラベル |
| 役割 | 再訪のきっかけ、鮮度感 |

**設計意図**:
- 代表メニューより上に配置
- 理由: 再訪者が「今月は何？」をすぐ確認できる
- 「動きがあるサイト」に見える効果

#### セクション4: 代表メニュー

| 項目 | 内容 |
|------|------|
| 内容 | 定番餃子、定食、スープ餃子など3〜4品 |
| 表現 | カルーセル的（横スクロール） |
| トーン | シンプル、押し売り感なし |

**設計意図**:
- 縦に長くならずスマートに複数品を見せる
- パッと見はシンプルだが、調べたい人には情報がある構成

#### セクション5: 店舗情報フッター

| 項目 | 内容 |
|------|------|
| 店名ロゴ | あり |
| 営業時間 | テキストで記載（通常時間のみ） |
| 営業日カレンダー | **画像エリア（スクエア）** ※必須で用意 |
| 予約ボタン | 外部リンク（Square） |
| Googleマップボタン | 外部リンク |
| 住所 | テキスト |
| 電話番号 | タップで発信可能に |
| SNSリンク | Instagram等 |

**重要**: 営業日カレンダーの画像エリアは必ず確保すること。画像差し替えで更新する運用。

### 5.3 固定バー（常時表示・画面下部）

| ボタン | リンク先 | 種類 |
|--------|----------|------|
| メニュー | ページ内（メニューセクションへ） | ページ内リンク |
| 予約 | Square予約ページ | 外部リンク |
| 店舗情報 | ページ内（フッターへ） | ページ内リンク |
| 地図 | Googleマップ | 外部リンク |

**設計経緯**:
- 当初「今月の餃子」「代表メニュー」を分けていた
- 統合して「メニュー」1枠にし、予約を追加
- 理由: 固定バーは最重要アクションに絞るべき。予約は明確なゴール（来店につながる）

### 5.4 採用しなかった要素

| 要素 | 理由 |
|------|------|
| バーガーメニュー | シングルページ＋固定バーで導線は完結。すっきりした世界観に合う |
| お知らせバー（ぐるぐる回るテキスト） | 読みにくい、落ち着いたトーンに合わない |
| SNS埋め込みセクション | お知らせ程度なら営業日カレンダー画像で対応可能。優先度低 |

## 6. 運用方針

### 6.1 情報の管理方針

| 情報 | 管理場所 |
|------|----------|
| 通常営業時間 | HP（スタティック）+ GBP |
| 営業日カレンダー | HP（画像差し替え） |
| 臨時休業・変更 | GBP中心（HPは画像更新で対応） |
| 今月の餃子 | HP（セクション更新） |
| 日々の投稿・告知 | Instagram |

### 6.2 更新頻度の想定

- **毎月更新**: 今月の餃子、営業日カレンダー画像
- **ほぼ更新なし**: 他のセクション（スタティック）
